v2.18.0
November 12, 2025
backend

Span diff and budget-aware sampling

You can now pin two traces side by side and Spanforge highlights exactly where their structure and timing diverge — the fastest way to explain a regression to a teammate.

  • Added Span diff in the explorer: pin a healthy trace against a slow one and see per-span deltas.
  • Added Budget-aware baseline sampling — rates now auto-tighten as a service nears its ingest ceiling.
  • Changed Default zstd compression for the OTLP exporter, ~22% smaller payloads on the wire.
  • Fixed Rare reordering of sibling spans with identical start timestamps in the waterfall.
v2.17.0
October 28, 2025
console

Deploy markers and 13-month retention GA

Overlay every release on your latency charts and scrub back through more than a year of hot traces. 13-month retention is now generally available on Team and Enterprise.

  • Added Deploy markers from GitHub Actions, GitLab CI and Argo CD via the /v2/deploys endpoint.
  • Added 13-month hot retention, GA — no cold restore step, fully queryable.
  • Changed SpanQL percentiles() now accepts up to eight thresholds in one call.
  • Fixed Service map degradation halos lingering after a service recovered.
v2.16.0
October 9, 2025
sdk

Rust SDK beta and faster Java agent

Our Rust distribution wrapping the tracing crate is now in public beta, with auto-instrumentation for axum, tonic and sqlx.

  • Added Rust SDK v0.24 (beta): cargo add spanforge-otel.
  • Changed Java agent startup overhead cut 40% by lazy-loading instrumentation modules.
  • Changed Node SDK now captures Prisma query spans with parameterized statement text.
  • Fixed Python Celery context propagation dropped across chord callbacks.
v2.15.0
September 22, 2025
alerting

Exemplar-linked alerts

Alerts now carry the actual traces that fired them. Open a PagerDuty incident and the five slowest exemplars are one click away — no more reconstructing the moment by hand.

  • Added Exemplar attachments on every alert, routed to PagerDuty, Opsgenie, Slack and Teams.
  • Added SLO burn-rate alerts with fast- and slow-burn windows.
  • Fixed Duplicate notifications when an alert flapped across the threshold within one minute.
v2.14.0
September 3, 2025
backend

Sub-second queries over 10B spans

A new columnar index and bloom-filtered attribute lookups bring p95 explorer query latency under 900ms even on the largest workspaces.

  • Changed Rebuilt the trace store on a columnar segment format with per-attribute bloom filters.
  • Changed p95 query latency over 10B spans dropped from 3.1s to 870ms.
  • Fixed High-cardinality tenant.id filters occasionally timing out on cold segments.
v2.13.0
August 14, 2025
security

SCIM provisioning and audit export

Enterprise workspaces can now sync users and groups via SCIM and stream the full audit log to their own SIEM.

  • Added SCIM 2.0 user and group provisioning for Okta and Entra ID.
  • Added Audit log export to S3, GCS and any OTLP-compatible SIEM.
  • Fixed Scoped API keys could read service names outside their assigned project.
