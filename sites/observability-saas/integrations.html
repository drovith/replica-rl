Integrations & SDKs

If it speaks OTLP, it speaks Spanforge.

There's no proprietary agent to adopt. Use the OpenTelemetry SDK for your language, add our exporter, and you're streaming traces. Below are the languages, frameworks and infra we auto-instrument out of the box.

60-second start

One endpoint, one key.

Already running an OpenTelemetry Collector? Add a single exporter block and restart. Every service already pushing OTLP lights up in the explorer immediately.

otel-collector.yamlCollector
exporters:
  otlp/spanforge:
    endpoint: ingest.spanforge.dev:4317
    headers:
      sf-api-key: ${SF_API_KEY}
    compression: zstd

service:
  pipelines:
    traces:
      receivers:  [otlp]
      exporters:  [otlp/spanforge]

Infrastructure

Context that survives the network.

Tracing only works if context crosses queues, proxies and databases. These integrations propagate it for you.

Kubernetes

Operator-managed collector DaemonSet with pod, node and namespace resource attributes auto-attached.

Kafka

Producer and consumer interceptors inject and extract traceparent on every message header.

PostgreSQL

Statement-level spans with db.statement, row counts and lock-wait timing captured.

Envoy / Istio

Mesh sidecars emit spans natively; Spanforge stitches them into your app traces seamlessly.

AWS SQS & SNS

Message-attribute propagation keeps async fan-outs connected to the originating request.

Redis

Command spans with key cardinality and round-trip latency, plus pipeline grouping.

gRPC

Bidirectional interceptors for unary and streaming RPCs across every supported language.

NGINX

OpenTelemetry module adds an edge root span and forwards context to upstreams.

Outbound & alerting

Plugs into the tools you already page from.

Spanforge fits your incident workflow rather than replacing it. Route alerts where your team already lives, and forward raw spans wherever you need them.

  • →PagerDuty & Opsgenie — alerts carry exemplar trace links straight into the incident.
  • →Slack & Teams — budget warnings and SLO burns posted with one-click trace deep links.
  • →Grafana — query Spanforge as a native data source via our PromQL-compatible API.
  • →S3 & GCS export — stream raw OTLP to your own bucket for archival or compliance.
alert-route.yamlRouting
alert: checkout-p99-burn
when:  p99("/v2/checkout") > 300ms for 5m
notify:
  - pagerduty: payments-oncall
  - slack:     "#sre-checkout"
attach:
  exemplars: 5   # slowest traces, linked
